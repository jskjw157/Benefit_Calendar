# 혜택 캘린더 (BenefitCal) MVP 프론트/백엔드 작업 분해 체크리스트

> 목적: MVP 실행을 위한 프론트/백엔드 작업을 기능 단위로 분해하여 병렬 진행을 돕는다.
>
> 범위: MVP 핵심 사용자 여정(온보딩 → 탐색 → 상세 → 캘린더 → 내 혜택 → 알림 → 대시보드) 구현

## 1) 공통/사전 합의
- [ ] API Base URL, 인증 방식(JWT) 확정
- [ ] API 명세 확정 및 변경 이력 관리 규칙 수립
- [ ] 공통 응답 포맷(success/error) 확정
- [ ] 도메인 용어 정의(User, Benefit, UserBenefit)
- [ ] 상태 코드/Enum 확정(UserBenefit, 신청 가능 상태, 카테고리, 지역 코드)
- [ ] 샘플 데이터 스키마/필드 정의 확정 및 공유
- [ ] 환경 변수/로컬 개발 가이드 정리

---

## 2) 프론트엔드 체크리스트

### 2.1 공통 레이아웃/기반 구성
- [ ] 글로벌 레이아웃/네비게이션 구조 확정
- [ ] 디자인 토큰(Tailwind 컬러/타이포) 고정
- [ ] 라우팅 구조 확정(홈/탐색/상세/캘린더/내 혜택/설정)
- [ ] 공통 컴포넌트 정리(버튼, 카드, 필터칩 등)
- [ ] 빈 상태/에러 상태 컴포넌트 준비

### 2.2 인증/프로필
- [ ] 로그인 화면 및 토큰 저장 처리
- [ ] 내 프로필 조회/수정 UI
- [ ] 프로필 완료 플로우(온보딩) 화면

### 2.3 혜택 탐색
- [ ] 검색 입력 + 디바운스/엔터 처리
- [ ] 필터(카테고리/지역/신청 가능) UI
- [ ] 정렬(마감일/최신순) UI
- [ ] 혜택 목록 카드 렌더링
- [ ] 페이지네이션/무한 스크롤 처리

### 2.4 혜택 상세
- [ ] 상세 화면 라우팅(benefitId)
- [ ] 기본 정보(기관/금액/기간) 노출
- [ ] 자격 요건/제출 서류 섹션
- [ ] 외부 신청 링크 버튼
- [ ] 북마크 버튼 및 상태 표시

### 2.5 캘린더
- [ ] 월간 캘린더 그리드 구현
- [ ] 날짜별 혜택 수 표시
- [ ] 날짜 클릭 시 혜택 리스트 노출
- [ ] 마감 임박 강조 표시

### 2.6 내 혜택
- [ ] 탭(북마크/준비중/신청완료/수령완료) UI
- [ ] 상태 변경 액션(드롭다운/버튼)
- [ ] 북마크 추가/해제 토글
- [ ] 빈 상태/에러 상태 처리

### 2.7 알림/설정
- [ ] 알림 설정 화면 UI(채널/마감 n일 전)
- [ ] 알림 설정 API 연동
- [ ] 알림 활성화/비활성화 토글

### 2.8 대시보드
- [ ] 요약 통계 위젯(매칭/마감 임박/신청 완료)
- [ ] 추천 혜택 리스트 연동

## 3) 백엔드 체크리스트

### 3.1 프로젝트 기반 구성
- [ ] 프로젝트 구조/환경 설정
- [ ] 인증(JWT) 기본 구현
- [ ] 에러 핸들링/유효성 검사 체계 구축

### 3.2 인증/사용자
- [ ] 로그인 API (JWT 발급)
- [ ] 사용자 프로필 조회/수정 API
- [ ] 사용자 스키마 및 마이그레이션

### 3.3 혜택 데이터
- [ ] Benefit 스키마 및 마이그레이션
- [ ] 혜택 목록 API (검색/필터/정렬/페이징)
- [ ] 혜택 상세 API
- [ ] 신청 가능 상태 계산 로직
- [ ] 샘플 데이터 시드 스크립트

### 3.4 사용자-혜택(UserBenefit)
- [ ] UserBenefit 스키마 및 마이그레이션
- [ ] 내 혜택 목록 API (상태 필터)
- [ ] 북마크 추가/해제 API
- [ ] 상태 변경 API

### 3.5 알림

- [ ] 이메일 발송 모듈(초기 1채널)

### 3.6 대시보드
- [ ] 요약 통계 API (매칭/마감 임박/신청 완료)
- [ ] 성능 점검(목록 조회 성능 최적화)

### 3.7 운영/품질
- [ ] 요청/응답 로깅
- [ ] 기본 모니터링 지표 수집
