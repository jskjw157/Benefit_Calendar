# 혜택 캘린더 (BenefitCal) MVP 프론트/백엔드 작업 분해 체크리스트

> 범위: MVP 핵심 사용자 여정(온보딩 → 탐색 → 상세 → 내 혜택 → 알림) 구현을 위한 작업 분해.

## 1) 공통 선행 작업
- [ ] API 명세 확정 및 변경 이력 관리 규칙 수립
- [ ] 샘플 데이터 스키마/필드 정의 확정
- [ ] 오류 코드/응답 포맷 통일
- [ ] 공통 용어 정의(상태 값, 카테고리, 지역 코드 등)

## 2) 프론트엔드 체크리스트

### 2.1 프로젝트 기반 구성
- [ ] 레이아웃(헤더/네비게이션) 공통 컴포넌트 정리
- [ ] 디자인 토큰(Tailwind 컬러/타이포) 고정
- [ ] 라우팅 구조 확정(홈/탐색/상세/캘린더/내 혜택/설정)

### 2.2 온보딩/프로필
- [ ] 프로필 입력 화면 UI
- [ ] 프로필 저장 API 연동
- [ ] 입력 검증 및 에러 메시지 처리

### 2.3 혜택 탐색
- [ ] 검색 바 동작(키워드 검색)
- [ ] 필터 UI(카테고리/지역/상태)
- [ ] 정렬 옵션 적용
- [ ] 페이지네이션/무한스크롤 적용
- [ ] 빈 상태/에러 상태 디자인

### 2.4 혜택 상세
- [ ] 상세 페이지 레이아웃 구성
- [ ] 신청 링크 CTA 버튼 구현
- [ ] 제출 서류 리스트 노출
- [ ] 북마크 버튼 및 상태 표시

### 2.5 캘린더
- [ ] 월간 그리드 렌더링
- [ ] 날짜 클릭 시 해당 혜택 리스트 표시
- [ ] 마감 임박 강조 표시

### 2.6 내 혜택
- [ ] 상태 탭 UI(북마크/준비중/신청완료/수령완료)
- [ ] 상태 변경 UI 연동
- [ ] 빈 상태/에러 상태 처리

### 2.7 알림/설정
- [ ] 알림 설정 화면 UI
- [ ] 알림 설정 API 연동
- [ ] 알림 상태 토글

## 3) 백엔드 체크리스트

### 3.1 프로젝트 기반 구성
- [ ] 프로젝트 구조/환경 설정
- [ ] DB 스키마(User/Benefit/UserBenefit) 확정

### 3.3 혜택 탐색

### 3.5 내 혜택

### 3.6 알림
